###2系统方案设计：

#####2.1系统总体架构：整体分为硬件底层层、驱动抽象层、FreeRTOS 内核层业务任务层四层架构。

1\. 硬件底层层（硬件电路）

对应需求：MCU、OLED、陀螺仪 MPU6050、电源管理

1.  主控单元：STM32F103C8T6（低成本穿戴 MCU）最小系统
2.  显示外设：0.96 寸 I2C OLED 屏幕
3.  传感外设：MPU6050 六轴陀螺仪加速度计（I2C 通信）
4.  电源模块：LDO 降压、锂电池充放电管理、低功耗休眠电路
5.  通信总线：I2C（OLED + 陀螺仪）、GPIO 通用输入输出

2\. 硬件驱动抽象层（HAL 驱动）

封装外设底层接口，解耦硬件与操作系统任务

- OLED 驱动：屏幕刷新、字符 / 数字显示、清屏、分页绘制 API
- MPU6050 驱动：加速度、角速度原始数据读取、寄存器配置
- 电源驱动：低功耗休眠、电压检测
- 系统时钟驱动：SysTick 定时器（FreeRTOS 系统节拍时钟源）

3\. FreeRTOS 内核支撑层（操作系统核心）

1.  任务调度模块
    - 抢占式调度，划分多优先级任务；4 种任务状态（运行 / 就绪 / 阻塞 / 挂起）
    - SysTick 滴答中断提供 1ms 系统时基，驱动任务切换
2.  任务间通信模块（解决共享资源冲突）
    - 二值信号量：传感器数据读取同步
    - 消息队列：OLED 界面数据缓存（时间、传感数据分发）
    - 互斥信号量：保护 I2C 总线（OLED / 陀螺仪共用 I2C，防止总线抢占冲突）
3.  内存管理模块
    - 采用 heap_4 内存分配策略，自动合并空闲内存块，减少穿戴设备内存碎片
4.  系统工具：延时函数、任务挂起 / 恢复、软件定时器（计时时间）

4\. 业务多任务层（核心业务，FreeRTOS 创建独立任务）

基于优先级划分 3 个基础任务，配合通信机制协同运行

1.  任务 1：时间更新任务（高优先级）
    - 功能：软件定时器计时，实时更新时分秒时间数据，通过消息队列发送时间数据
    - 阻塞方式：vTaskDelay 周期性延时
2.  任务 2：陀螺仪传感采集任务（中优先级）
    - 功能：周期性读取 MPU6050 原始运动数据，数据滤波处理，通过消息队列上传传感数据
    - 同步：二值信号量控制采集周期，互斥锁占用 I2C 总线
3.  任务 3：OLED 界面刷新任务（低优先级）
    - 功能：监听消息队列，接收时间 / 传感数据，切换绘制页面（时间页面、传感器数据页面）
    - 资源保护：互斥信号量独占 OLED 显示硬件

**详细设计文档**

#####3.1硬件电路原理图

一．STM32F103C8T6 最小系统原理图

1.  电源引脚

VDD (3.3V)、VSS (GND)，每组电源引脚就近并联 0.1uF 去耦电容；

BOOT0 引脚通过 10K 下拉电阻接 GND（主闪存启动模式）。

1.  复位电路

NRST 引脚：上拉 10K 电阻至 3.3V，并联 104 电容到 GND，串联轻触按键到 GND，上电自动复位 + 手动按键复位。

1.  外部高速晶振 HSE

OSC_IN (PD0)、OSC_OUT (PD1) 接 8MHz 无源晶振，两端各 22pF 电容到 GND，为主控提供系统主时钟。

1.  下载调试接口（SWD）

PA13 (SWDIO)、PA14 (SWCLK) 引出 2.54 排针，预留下载调试。

1.  I2C 通信引脚（共用总线）

PB6 → I2C_SCL，PB7 → I2C_SDA；各上拉 4.7K 电阻至 3.3V，匹配 I2C 开漏通信电平。

1.  预留扩展 GPIO（扩展功能备用）

PA0、PA1、PA2、PA3 引出排针，用于后期接旋转编码器、蓝牙模块。

1.  SysTick：芯片内部集成，无需外部电路，由内核配置生成 1ms 滴答中断。

二、模块 3：MPU6050 六轴陀螺仪传感器电路

1.  供电：VCC 接系统 3.3V，GND 公共地；
2.  I2C 总线：SDA 接主控 PB7，SCL 接主控 PB6；
3.  AD0 引脚：10K 下拉 GND，I2C 设备地址固定 0x68；若上拉 3.3V 地址变为 0x69；
4.  INT 中断引脚：预留接 PA0（软件可选，基础版不使用中断，仅轮询读取）。

三、模块 4：0.96 寸 I2C OLED 显示屏幕电路

1.  供电：VCC 3.3V，GND 公共地；严禁 5V 供电，屏幕会烧毁；
2.  通信：SDA 接 PB7，SCL 接 PB6，与 MPU6050 共享同一组 I2C 总线；
3.  RES 复位引脚：串联 10K 上拉至 3.3V，也可直接接 MCU 任意 GPIO 做软件复位（推荐 PA1）；
4.  DC/SA0 引脚：下拉 GND，固定 I2C 通信地址。

**模块划分**

**一、底层硬件驱动模块（板级驱动，无操作系统依赖）**

**作用：封装寄存器操作，提供标准化硬件调用接口，隔离硬件与上层业务**

1.  **系统时钟 & SysTick 驱动模块**
    - **功能：配置 HSE 外部晶振、系统主时钟分频；初始化 SysTick 定时器，为 FreeRTOS 提供 1ms 系统节拍时基**
    - **对外接口：SysClock_Init()、SysTick_Init()**
2.  **I2C 总线驱动模块**
    - **功能：硬件 I2C 初始化、I2C 读写单字节 / 多字节通用函数，统一供给 OLED、MPU6050 调用**
    - **对外接口：I2C_Init()、I2C_WriteData()、I2C_ReadData()**
3.  **OLED 屏幕驱动模块**
    - **功能：屏幕初始化、清屏、字符 / 数字显示、坐标打点、整屏缓冲区刷新**
    - **对外接口：OLED_Init()、OLED_ShowNum()、OLED_Clear()、OLED_Refresh()**
4.  **MPU6050 陀螺仪驱动模块**
    - **功能：传感器寄存器初始化、读取三轴加速度 / 角速度原始数据、基础均值滤波**
    - **对外接口：MPU6050_Init()、MPU6050_GetData()**
5.  **电源管理驱动模块**
    - **功能：电池电压采集、MCU 休眠 / 唤醒底层配置**
    - **对外接口：Power_Init()、MCU_LowPower_Sleep()**

**二、FreeRTOS 内核支撑模块（操作系统基础服务）**

**作用：封装 FreeRTOS 原生 API，统一管理调度、通信、内存，为业务任务提供 OS 能力**

1.  **内存管理模块**
    - **配置：启用heap_4内存分配方案，实现空闲内存块排序、相邻块合并，降低内存碎片**
    - **功能：动态内存申请释放、内存碎片检测接口**
2.  **任务通信同步模块**
    - **资源：创建 I2C 互斥信号量、采集同步二值信号量、时间消息队列、传感器数据消息队列**
    - **封装接口：队列读写、信号量获取 / 释放统一函数，简化上层调用**
3.  **任务基础管理模块**
    - **功能：任务创建、优先级配置、延时、挂起恢复封装；空闲任务钩子函数（低功耗休眠入口）**
4.  **系统定时器模块**
    - **功能：FreeRTOS 软件定时器封装，提供秒级计时基础能力，支撑时间任务计时**

**三、业务功能模块（业务逻辑，独立解耦，被任务调用）**

1.  **系统计时模块**
    - **功能：时分秒计时逻辑、时间进位处理、时间数据结构体打包**
    - **输入：软件定时器周期信号；输出：标准时间数据包，推送至时间消息队列**
2.  **运动传感处理模块**
    - **功能：接收 MPU6050 原始数据、数字滤波、运动数据结构体封装**
    - **输入：传感器原始采样值；输出：滤波后三轴加速度、角速度数据，推送至传感队列**
3.  **共享资源保护模块**
    - **功能：封装 I2C 总线上锁、释放操作，统一调用互斥信号量，避免重复写信号量代码**

###四、FreeRTOS 应用任务模块（系统并发运行主体，多优先级）**

**基于抢占式调度划分 3 个独立任务，每个任务循环调用对应业务模块**

1.  **Time_Task 时间更新任务（高优先级）**

**调用模块：系统计时模块、消息队列通信接口**

**逻辑：每秒更新时间，打包数据写入时间消息队列，vTaskDelay 阻塞**

1.  **Sensor_Task 陀螺仪采集任务（中优先级）**

**调用模块：MPU6050 驱动、运动传感处理模块、共享资源保护模块**

**逻辑：周期采集传感器数据，滤波后存入传感消息队列**

1.  **OLED_Display_Task 界面刷新任务（低优先级）**

**调用模块：OLED 驱动、共享资源保护模块、界面逻辑模块**

**逻辑：阻塞等待队列数据，获取 I2C 总线后绘制对应页面**

###五、界面逻辑模块（上层显示业务）**

1.  **时间页面绘制子模块**
    - **输入：时间数据包；输出：调用 OLED 驱动，在屏幕指定坐标显示时、分、秒**
2.  **传感器数据页面绘制子模块**
    - **输入：滤波后运动数据；输出：调用 OLED 驱动，打印三轴加速度、角速度数值**
3.  **页面切换控制子模块**
    - **基础版：固定循环切换时间 / 传感双页面；预留接口对接扩展旋转编码器菜单逻辑**

**3.3 外设配置方案**

**所有外设均通过 STM32CubeMX 配置，核心参数如下：**

**（1）系统时钟配置**

- **时钟源：外部 8MHz 晶振（HSE）**
- **PLL 倍频：×9，输出 72MHz 作为系统时钟**
- **AHB 预分频：/1（72MHz）**
- **APB1 预分频：/2（36MHz）**
- **APB2 预分频：/1（72MHz）**
- **时基：启用 FreeRTOS 后切换为 TIM1，避免与 SysTick 冲突**

**（2）I2C1 配置**

- **引脚：PB6-SCL、PB7-SDA**
- **速度模式：Fast Mode**
- **时钟频率：400kHz**
- **地址模式：7 位从机地址**
- **其余参数保持默认**

**（3）USART1 配置**

- **引脚：PA9-TX、PA10-RX**
- **波特率：115200**
- **数据位：8 位**
- **停止位：1 位**
- **校验位：无**
- **DMA：开启 RX 通道 DMA1_Channel5，工作在环形模式**
- **中断：开启全局中断**

**（4）定时器配置**

- **TIM2（秒计时基准）：预分频器 7199，计数周期 9999，计数频率 10kHz，1 秒触发一次更新中断**
- **TIM3（编码器模式）：双通道编码器模式，计数范围 0~65535，预分频器 0，硬件自动识别旋转方向与计数**

**（5）中断优先级配置（FreeRTOS 适配）**

**按照抢占优先级从高到低排序：**

- **TIM2（计时）：抢占优先级 5，子优先级 0**
- **I2C1 事件：抢占优先级 6，子优先级 0**
- **I2C1 错误：抢占优先级 6，子优先级 0**
- **USART1 全局：抢占优先级 7，子优先级 0**
- **EXTI10（按键）：抢占优先级 7，子优先级 0**
- **SysTick：抢占优先级 15，子优先级 0**
